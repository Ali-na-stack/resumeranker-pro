

## Fix "View Original Resume" Link

### Problem
The "View Original Resume" button links to a public URL generated by `getPublicUrl()`, but the `resumes` storage bucket is likely not public — so the URL leads to an invalid/forbidden page.

### Solution
Instead of storing a static public URL at upload time, generate a **signed URL** on-demand when the user clicks the button. This works regardless of bucket visibility.

### Changes

**1. `src/lib/api.ts`**
- Change `uploadResumeFile` to return just the **file path** (e.g. `jobId/timestamp_file.pdf`) instead of a public URL
- Add a new `getResumeSignedUrl(filePath: string)` function that creates a time-limited signed URL using `supabase.storage.from("resumes").createSignedUrl(filePath, 3600)` (1 hour expiry)

**2. `src/pages/CandidateDetail.tsx`**
- When user clicks "View Original Resume", call `getResumeSignedUrl(candidate.resume_url)` to get a working signed URL, then open it in a new tab
- Add a small loading state on the button while the URL is being generated

### Why this works
- Signed URLs work for both public and private buckets
- The stored `resume_url` becomes a path reference rather than a potentially broken public URL
- Existing records that already have full public URLs will need the path extracted, which we'll handle gracefully
